# UI/UX Transformation Plan — Vitals Dashboard

**Date:** 2026-03-17
**Status:** Research Complete — Decisions Confirmed
**Branch:** `experiment/ui-ux-pro-max`
**Tool:** UI/UX Pro Max design intelligence skill

---

## Executive Summary

The current Vitals UI is functional but has significant UX issues identified from live screenshots. This plan transforms it into a **data-driven, information-centric, mobile-first** health dashboard.

---

## 0. Current State Analysis (From Live Screenshots)

### Dashboard — Desktop
- **KPI cards are oversized**: 3 cards (2195 kcal, 2 sessions, 66.9 kg) take up ~40% of the left column but show only 3 numbers — poor data density
- **AI Report dominates**: The report panel consumes the entire left column below KPIs. Action items with priority badges are useful but buried below a long summary paragraph
- **Charts are basic**: Nutrition trends chart shows 4 flat lines (calories ~2200, others near 0-300) — the y-axis scale (0-2400) makes protein/carbs/fat lines look flat and unreadable
- **Workout volume chart**: Only 2 bars visible (Mar 14, Mar 15) — good data but visually sparse
- **Body weight chart**: Clean line from 66.3-67.5 kg — works well, good use of tight y-axis

### Dashboard — Mobile (CRITICAL ISSUES)
- **KPI cards are full-width**: Each card takes an entire screen height — user must scroll 3+ screens to see charts
- **AI Report pushes everything down**: The report text is long and untruncated — charts are 5+ scrolls away
- **No bottom navigation**: hamburger menu requires opening drawer, then tapping — 2 actions to navigate
- **No quick actions visible**: Upload Data is hidden in sidebar drawer

### Nutrition — Desktop
- **Pie chart is too large**: Takes 50% of viewport to show 3 numbers (Protein: 648, Carbs: 1110, Fat: 492)
- **Table is functional**: Clean daily rows with all macros — but no visual indicators (no color, no bars, no trend arrows)
- **No target comparison**: Numbers shown but no indication of whether they're good/bad/on-track

### Nutrition — Mobile
- **Pie chart still huge**: Fills entire first screen
- **Table columns truncated**: Fat and Fiber columns are cut off on 390px viewport
- **No horizontal scroll indicator**: User may not know there's more data

### Workouts — Desktop
- **Exercise selector defaults to "Elliptical Trainer"**: Shows a flat line at 0 — terrible first impression
- **Progress chart takes 50% of page**: For a flat line with no meaningful data
- **Session cards are good**: Title, duration, volume, set count, exercise badges — but badges wrap and look cluttered

### Workouts — Mobile
- **Same issues magnified**: Flat line chart takes entire screen before session cards appear
- **Exercise badges wrap to 3+ lines**: On mobile the badges take more space than the actual session info

### Reports — Desktop & Mobile
- **Wall of text**: Long AI summaries dominate — no visual hierarchy beyond date headers
- **Coverage badges are good**: "6d nutrition", "4d workouts", "3d biometrics" — but buried in text
- **No scorecard visible in list view**: Must expand to see scores
- **No trend between reports**: Can't see if things are improving week over week

## 1. Design System (Generated by UI/UX Pro Max)

### Style: Data-Dense Dashboard
- Multiple charts/widgets, KPI cards, minimal padding, grid layout, space-efficient
- Best for: business intelligence, analytics, operational dashboards
- Performance: Excellent | Accessibility: WCAG AA

### Color Palette (Hybrid: Healthcare + Analytics)

| Role | Hex | Usage |
|------|-----|-------|
| Primary | `#0891B2` | Navigation, headers, primary actions (healthcare cyan) |
| Secondary | `#3B82F6` | Chart lines, links, secondary elements |
| Accent/CTA | `#F97316` | Call-to-action buttons, alerts, important highlights |
| Success | `#059669` | Positive trends, good scores, on-target metrics |
| Warning | `#D97706` | Medium priority, approaching limits |
| Destructive | `#DC2626` | Negative trends, bad scores, high priority |
| Background | `#F8FAFC` | Page background |
| Card | `#FFFFFF` | Card surfaces |
| Text | `#1E293B` | Primary text |
| Muted | `#64748B` | Secondary text, labels |
| Border | `#DBEAFE` | Card borders, dividers |

**Chart-specific colors:**
- Calories: `#F97316` (orange)
- Protein: `#3B82F6` (blue)
- Carbs: `#EAB308` (yellow)
- Fat: `#EF4444` (red)
- Fiber: `#22C55E` (green)
- Weight: `#8B5CF6` (purple)
- Volume: `#06B6D4` (cyan)

### Typography: JetBrains Mono Variable (Single Font)

| Element | Font | Weight | Size |
|---------|------|--------|------|
| KPI numbers | JetBrains Mono | 600 | 28-36px |
| Headings | JetBrains Mono | 500-600 | 18-24px |
| Body text | JetBrains Mono | 400 | 14-16px |
| Labels/captions | JetBrains Mono | 500 | 12-13px |
| Table data | JetBrains Mono | 400 | 13-14px (tabular nums) |

**Why JetBrains Mono for everything:** Single variable font = one download, consistent visual identity across the entire app. Tabular figures align data columns, monospace makes numbers scannable, and the technical feel matches a health analytics product. Variable font axes allow fine-tuning weights without extra files.

```css
@import url('https://fonts.googleapis.com/css2?family=JetBrains+Mono:ital,wght@0,100..800;1,100..800&display=swap');
```

## 2. Page-by-Page Improvement Plan

### 2.1 Dashboard (Priority: HIGHEST)

**Current problems:**
- KPI cards are too large with too little data
- Report panel takes 50% of desktop width but is rarely the first thing you check
- Charts are generic with no trend indicators
- No "at a glance" health score

**Proposed changes:**

#### A. Hero KPI Strip (Top of page)
Replace the large weekly summary card with a compact, dense KPI strip:

```
┌──────────┬──────────┬──────────┬──────────┬──────────┐
│ 2,150    │ 4        │ 78.3 kg  │ 156g     │ 7.2/10   │
│ avg cal  │ sessions │ weight   │ protein  │ AI score │
│ ▲ +120   │ ▼ -1     │ → stable │ ▲ +12g   │ ▲ +0.4   │
└──────────┴──────────┴──────────┴──────────┴──────────┘
```

- Large monospace number (Fira Code 600, 28px)
- Small label below (Fira Sans 500, 12px, muted)
- Trend indicator with color: green ▲, red ▼, gray →
- Delta from previous period
- Mobile: horizontal scroll or 2x3 grid

#### B. Spark Charts Below KPIs
Tiny inline sparklines (40px height) under each KPI showing the 7-day trend. No axis labels, just the shape.

#### C. Bento Grid Layout for Charts
Replace the linear card stack with a bento grid:

```
Desktop (1440px):
┌────────────────────┬──────────────┐
│ Nutrition Trend     │ Weight       │
│ (2/3 width)        │ (1/3)        │
├──────────┬─────────┼──────────────┤
│ Workout  │ Macro   │ Activity     │
│ Volume   │ Split   │ Heatmap      │
│ (1/3)    │ (1/3)   │ (1/3)        │
└──────────┴─────────┴──────────────┘
```

#### D. Report Summary → Inline Alert
Instead of a full report panel, show a compact alert bar:
```
┌─────────────────────────────────────────────┐
│ 📊 Weekly Report: "Protein up, sleep down"  │
│ Score: 7.2/10  •  3 action items  [View →]  │
└─────────────────────────────────────────────┘
```
Full report moves to the Reports page.

#### E. NEW: Activity Heatmap
Add a GitHub-style calendar heatmap showing workout days (green) and rest days (gray) for the past 90 days. Glanceable at-a-glance consistency tracker.

### 2.2 Nutrition Page

**Current problems:**
- Pie chart is static and doesn't show targets
- Table has no conditional formatting
- No daily goals or progress bars

**Proposed changes:**

#### A. Macro Target Rings (Replace Pie Chart)
Three concentric progress rings showing % of daily target:
- Outer: Calories (orange)
- Middle: Protein (blue)
- Inner: Carbs/Fat (yellow/red)
- Center: "1,850 / 2,200 kcal" text

#### B. Daily Nutrition Table → Data Grid
- Add conditional coloring: green if protein > 140g, red if < 100g
- Add mini bar charts in cells for visual comparison
- Sticky header with column sorting
- Row highlighting on hover

#### C. NEW: Nutrient Trend Sparklines
Add 7-day sparklines next to each macro header showing the trend direction.

### 2.3 Workouts Page

**Current problems:**
- Session cards are text-heavy with no visual hierarchy
- Exercise progress chart needs context (targets, PRs)
- No workout frequency visualization

**Proposed changes:**

#### A. Session Cards → Compact Timeline
Replace full-width cards with a compact vertical timeline:
```
Mon 14 ──── Push Day ──── 12,450 kg ──── 28 sets
             Bench: 80kg×8 (PR!)
Wed 12 ──── Pull Day ──── 9,800 kg ──── 24 sets
```
- Date on left, title center, volume right
- Best set highlighted per exercise
- PR badges (personal record markers)

#### B. Exercise Progress Chart Enhancements
- Add target/goal line (dashed)
- Highlight PR points with special markers
- Show volume trend (secondary axis)
- Add forecasting band (light shaded area projecting next 2 weeks)

#### C. NEW: Weekly Volume Bar (Muscle Group Split)
Stacked bar chart showing volume per muscle group per week. Helps identify imbalances.

### 2.4 Reports Page

**Current problems:**
- Report cards are expand-only, no quick summary
- No comparison between reports
- Action items are buried

**Proposed changes:**

#### A. Report Scorecard Comparison
Show last 4 weekly scores as a mini trend line with the current score prominent.

#### B. Action Items → Kanban Board
Move action items to a simple 3-column layout:
- To Do | In Progress | Done
- Users can drag items (or tap to toggle on mobile)
- Persists state locally

#### C. Report Content → Tabbed Sections
Instead of long expandable content, use tabs:
`[Summary] [Scorecard] [Actions] [Deep Dive]`

## 3. Mobile-Specific Patterns

### 3.1 Bottom Navigation Bar
Replace the sidebar + hamburger menu with a fixed bottom nav (iOS-style tab bar):

```
┌──────────────────────────────────────┐
│  🏠       🥗       💪       📊      │
│ Home   Nutrition Workouts Reports    │
└──────────────────────────────────────┘
```

- 4 items max (within Material/Apple guidelines)
- Active state: filled icon + label + color
- Upload action: FAB (floating action button) or long-press on home tab

### 3.2 Thumb Zone Optimization
- Primary actions in bottom 1/3 of screen
- Date range picker: bottom sheet instead of top bar
- Charts: swipeable cards instead of scrolling down
- KPIs: top strip is glanceable, charts are below the fold

### 3.3 Pull-to-Refresh
Refresh dashboard data on pull-down gesture. Show skeleton loaders during refresh.

### 3.4 Swipeable Chart Cards
On mobile, charts become swipeable cards:
```
← [Nutrition] [Volume] [Weight] [Heatmap] →
   ●  ○  ○  ○  (dot indicators)
```

### 3.5 Compact Mobile Dashboard
```
┌────────────────────────┐
│ KPI Strip (horizontal  │
│ scroll: cal/sessions/  │
│ weight/protein/score)  │
├────────────────────────┤
│ ⚠ Report Alert Bar     │

│ ← Swipeable Charts →  │

│    ●  ○  ○  ○         │
├────────────────────────┤
│ Recent Workouts        │
│ (compact timeline)     │
```

## 4. Chart & Data Visualization Improvements

Based on UI/UX Pro Max chart recommendations:

| Current | Replace With | Why |
|---------|-------------|-----|
| Line chart (nutrition) | Line with confidence band | Show daily variation range, not just averages |
| Bar chart (workout volume) | Stacked bar (by muscle group) | More information per chart |
| Line chart (weight) | Line + target band | Show goal range visually |
| Pie chart (macros) | Progress rings or bullet chart | Better for target comparison |
| None | Calendar heatmap | Consistency/frequency at a glance |
| None | Bullet charts (KPI strip) | Compact KPI with target ranges |
| None | Sparklines (inline) | Trend direction in minimal space |

### Recharts-Compatible Additions
Since you already use Recharts:
- `RadialBarChart` for macro progress rings
- `AreaChart` with gradient fill for trend bands
- Custom `Sparkline` component (tiny LineChart, no axes)
- Heatmap: custom SVG or nivo/heatmap library
